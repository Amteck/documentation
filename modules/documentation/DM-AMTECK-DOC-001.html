Document Management - New Documentation Book in COINS

  • Item Name : Document Management - New Documentation Book in COINS
  • Item Version : v 1.0
  • Author : Nicholas Knight
  • License : GNU GPL

Books in the COINS documentation behave in a similar way to the menus in the rest of COINS OA, and in fact use the same system. A book can contain documentation topics and other books, in the same way that a COINS menu can contain runnable functions and other menus. If you want to create your own documentation book, you need to add it to the existing documentation menu structure by creating a user-defined copy of one of the documentation books, and adding your new book to this copy.

Table of Contents

  1. To add a new book

Add a New Book

  1. Go to System > System Setup > Help Tools > Topic Maintenance.
  2. Using the tree view, browse to the book that you want to add the new book to (we'll call this the parent book).
  3. Using the Add Type drop-down, select New menu book and either:
    1. Click ADD to add the book at the bottom of the list.
    2. Select an existing item and click INSERT to add the book above the selected item.
  4. Enter a function code for the new book. This can be any code you choose, but do not begin it with % or +.
  5. The module defaults from the module of the parent book; you can select a different one if you wish. The module shows on the Search results.
  6. Enter a description for the book, to appear in the documentation menu.
  7. Click SAVE.
  8. In the tree view, click next to the parent book; this shows the new book in the tree view.
  9. Click the new book in the tree view. You can now add topics or other books to the new book.
